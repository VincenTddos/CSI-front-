# Wi-Care 系統 Wi-Fi 三角定位參數設定指南

本文件介紹 `WiFi_Location2.py` 中與 Wi-Fi 三角定位相關的核心參數。這些參數會直接影響室內定位的準確度與穩定性，您可依據實際部署環境進行調校。

## 📍 基地台與座標設定

### `KNOWN_APS` / `KNOWN_DEVICES`
*   **用途**：定義系統已知的 Wi-Fi 基地台 (Access Points) 及其在實體空間中的相對座標。
*   **格式**：字典格式，鍵 (Key) 為基地台的 `SSID`（大小寫敏感），值 (Value) 為 `(x, y)` 座標（單位：公尺）。
*   **設定建議**：
    *   定位至少需要 **3 個**能接收到訊號的已知基地台（Anchor）。
    *   增加已知 AP 數量且將其平均分布於空間周圍，通常能提升定位穩定度與精準度。
*   **範例**：
    ```python
    KNOWN_APS = {
        "MyWiFi-2.4G": (0.0, 0.0),    # 原點
        "MyWiFi-5G": (6.0, 0.0),      # x 軸 6 公尺處
        "Office-AP": (3.0, 5.0),      # x 軸 3 公尺，y 軸 5 公尺處
        "Guest-Network": (3.0, 5.0),
    }
    ```

## 📶 訊號衰減模型參數 (Log-Distance Path Loss Model)

系統透過將接收到的訊號強度 (RSSI) 轉換為距離，這部分依賴於「對數距離路徑損耗模型」。校正以下兩個參數對於準確測距至關重要。

### `A_AT_1M`
*   **用途**：距離基地台 **1 公尺處**的基準訊號強度 (RSSI)，單位為 dBm。
*   **預設值**：`-45.0`
*   **設定建議**：使用手機或電腦在距離各已知 AP 確實 1 公尺處測量 RSSI 並取平均值。數值通常為負數（如 -40 到 -50 之間）。

### `PATH_LOSS_N`
*   **用途**：環境衰減係數 (Path Loss Exponent)，反映訊號隨距離衰減的速率。
*   **預設值**：`2.7`
*   **設定建議**：
    *   無障礙空曠空間：`2.0`
    *   一般室內環境（辦公室/居家）：`2.7` ~ `3.0`
    *   隔間多、障礙物密集的室內：`3.0` ~ `4.0` 或更高。
    *   **校正方式**：在已知距離（如 3 米、5 米處）測量實際 RSSI，並微調此係數直到系統算出的距離與實際距離吻合。

## 🔍 掃描與運算策略參數

Wi-Fi 訊號本身容易受到環境雜訊干擾而產生浮動，因此系統會進行多次掃描取平均來穩定數值。

### `SCAN_ROUNDS`
*   **用途**：每次產生一次定位結果前，要進行多少回合的 Wi-Fi 掃描。
*   **預設值**：`6`
*   **設定建議**：
    *   數值越高：收集到的樣本越多，平均值越穩定，定位座標跳動較小。但定位更新的延遲會變長。
    *   數值越低：定位更新反應快，但容易因單次訊號干擾造成座標劇烈跳動。

### `SCAN_INTERVAL_SEC`
*   **用途**：每次掃描回合之間的間隔時間（單位：秒）。
*   **預設值**：`0.7`
*   **設定建議**：保留適當間隔可避免網卡過載並取得不同時間點的訊號樣本。總定位耗時約為 `SCAN_ROUNDS` × `SCAN_INTERVAL_SEC`。

### `MAX_ANCHORS`
*   **用途**：限制每次多邊定位計算時，最多使用幾組訊號最強的基地台。
*   **預設值**：`None` (表示可用就全用)
*   **設定建議**：若環境中登錄了許多較遠的 AP，可以設定此值（例如 `5`）。系統會優先取 RSSI 最強（最接近）的 5 個 AP 進行計算，避免微弱且充滿雜訊的遠端 AP 訊號拉歪定位結果。

---

**如何優化定位準確度？**
1. 確認並更新 `KNOWN_APS` 中設定的實體座標是否與現場完全一致。
2. 針對您的環境重新測量並微調 `A_AT_1M` 與 `PATH_LOSS_N` 參數。
3. 確保測試設備的 Wi-Fi 網卡沒有被省電模式限制，這可能導致掃描到的 RSSI 不準確。
